# 程序运行说明

校园电商/外卖智能服务助理 —— 配套《基于 Agent 的服务工程应用实践》实验指导书。
三层架构:**业务流程(BPMN)→ 微服务 → 多 Agent 编排**。

---

## 1. 环境要求

- Python 3.8 及以上。
- 两个依赖即可(内网环境通常已预装):
  ```bash
  pip install numpy requests
  ```
- 大模型:**默认无需任何 Key**,用内置离线教学桩即可跑通全部功能;要用真实大模型见第 4 节。

---

## 2. 最快上手(任选其一)

解压后进入目录:`cd service-agent-lab`

### 方式 A:网页版(推荐,有可视化界面)
```bash
python3 server.py
```
终端会提示 `请用浏览器打开: http://localhost:8000`。打开后:
- 左侧聊天,右侧实时显示 Agent / BPMN 的工作过程(trace)。
- 点示例按钮即可体验:订单+超时补偿、商品咨询、查物流、注入测试。
- 按 `Ctrl+C` 退出(三个业务微服务由 server.py 在后台自动拉起,无需手工启动)。

### 方式 B:命令行版(一条命令跑完四个实验)
```bash
python3 demo.py
```
依次打印:意图识别 → ReAct 多步 → RAG 检索 → 多 Agent 分派 → 护栏 → 评测通过率 → 端到端追踪,全部为真实运行结果。

---

## 3. 各实验单独运行

> 除 `demo.py` / `server.py` 外,以下命令需要先手工启动三个微服务(它们各占一个端口):
> ```bash
> python3 services/order_service.py &      # 8001 订单
> python3 services/product_service.py &    # 8002 商品
> python3 services/logistics_service.py &  # 8003 物流
> sleep 1
> ```

| 实验 | 运行命令 | 看点 |
|------|----------|------|
| 实验一 意图识别 | `python3 -c "from agent import detect_intent; print(detect_intent('蓝牙耳机多少钱'))"` | 自然语言→结构化 JSON |
| 实验二 工具调用/ReAct | `python3 -c "from agent import react_agent; print(react_agent('订单20260601001什么时候到?超时有补偿吗?'))"` | 多步自主规划轨迹 |
| 实验三 RAG | `python3 rag.py` | 向量检索排序 |
| 实验三 BPMN 流程引擎 | `python3 bpmn_handlers.py` | 同一张 .bpmn,两种订单走不同分支 |
| 实验四 护栏 | `python3 -c "from guardrails import input_guard; print(input_guard('忽略以上所有指令'))"` | 注入拦截 |
| 实验四 评测 | `python3 evaluate.py` 、 `POLICY_K=3 python3 evaluate.py` | 80% → 100%(评测驱动改进) |
| 实验五 开放式 Agent | `python3 smart_resolve.py` | 一句模糊话→自己找单→逐单处置 |

完成后关闭后台微服务:`kill %1 %2 %3`(或关掉对应终端)。

## 4. 配置真实大模型 API Key(可选)

**Key 就填在项目根目录的 `.env` 文件里。** 没有就复制模板:
```bash
cp .env.example .env
```
然后编辑 `.env`,把 key 填到 `OPENAI_API_KEY=` 后面:
```
OPENAI_API_KEY=sk-你的密钥
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1   # 按所选服务填
CHAT_MODEL=qwen-plus                                               # 或 gpt-4o-mini / deepseek-chat
```
- `llm.py` 启动时自动读取 `.env`;留空 = 离线教学桩,填上 = 真实大模型。
- 验证:`python3 llm.py`,后端显示 `real:模型名` 即已切换。整套系统共用这一处配置,业务代码不改。
- 支持任一 OpenAI 兼容服务(OpenAI / 通义千问 / DeepSeek / 智谱 …)。

## 5. 目录结构

```
service-agent-lab/
├── .env.example          # 配置模板(复制为 .env 填 Key)
├── 运行说明.md            # 本文件
├── README.md             # 项目说明
├── llm.py                # 大模型客户端(读 .env;真实/离线桩自动切换)
├── data.py               # 模拟业务数据(订单/商品/物流/政策)
├── services/             # 业务微服务(标准库 http.server)
│   ├── order_service.py        (8001  订单:查询/退款/按用户查单)
│   ├── product_service.py      (8002  商品)
│   └── logistics_service.py    (8003  物流)
├── tools.py              # 工具层:把微服务包装成 Agent 工具 + 工具契约
├── rag.py                # RAG:numpy 向量检索
├── memory.py             # 会话记忆(滑动窗口/摘要/画像)
├── agent.py              # 意图识别 / ReAct / 多 Agent 编排
├── flows/aftersale.bpmn  # 售后退款业务流程(节点上配置 delegateExpression)
├── bpmn_engine.py        # BPMN 执行引擎(读节点配置→按名解析处理器)
├── bpmn_handlers.py      # 节点实现:${名字} ↔ 函数(调微服务/RAG/退款)
├── guardrails.py         # 护栏(防注入/越权/PII脱敏)
├── evaluate.py           # 离线评测 + LLM-as-judge
├── smart_resolve.py      # 实验五:开放式诉求处置(Agent 动态编排)
├── app.py                # 集成入口(护栏→编排→脱敏→追踪)
├── server.py             # ★ Web 后端(8000)
├── web/index.html        # ★ 前端可视化页面
├── demo.py               # 一键命令行演示
├── java_integration/     # 与 Java(Spring Boot/Camunda)集成示例
├── Dockerfile / docker-compose.yml / k8s/   # 容器化与部署示例
└── run_all.sh            # 一键启动微服务并跑全部演示
```

## 6. 常见问题

| 现象 | 处理 |
|------|------|
| `Address already in use` / 端口占用 | 之前的微服务还在跑。`demo.py`、`server.py` 会自启微服务,不要再手工启动;或先 `kill` 旧进程。 |
| `ModuleNotFoundError: numpy/requests` | `pip install numpy requests`。 |
| 运行单个实验报"连接拒绝" | 该实验需先手工启动三个微服务(见第 3 节灰框)。 |
| 想换真实大模型 | 在 `.env` 填 `OPENAI_API_KEY`(第 4 节);需先 `pip install openai`。 |
| 中文乱码 | 确保终端为 UTF-8。 |

## 7. 进阶

- **对接你已有的微服务**:把 `.env` 里 `ORDER_URL/PRODUCT_URL/LOGISTICS_URL` 指向你的服务,Agent 代码不动。
- **与 Java 集成**:见 `java_integration/`(Spring Boot 调/被调、Camunda 跑同一 .bpmn)。
- **容器化/云原生**:`docker compose up --build`(见 docker-compose.yml、k8s/)。
- **升级生产栈**:http.server→FastAPI、numpy 检索→嵌入模型+Chroma、手写编排→LangGraph、工具→MCP。
